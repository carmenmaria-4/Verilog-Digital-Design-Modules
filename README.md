# Digital Design Projects - Verilog Implementations

This repository contains a collection of digital logic design projects implemented in **Verilog/SystemVerilog** using **Vivado 2025.2**. The projects range from arithmetic units to hardware-level encryption and ROM-based lookup tables.

All projects are targeted for the **Xilinx Spartan-7 FPGA (Part: xc7s50csga324-1)**.

---

## 1. 8-Bit Arithmetic Logic Unit (ALU)
**Directory:** `/ALU`

A simplified ALU that performs calculations between two 8-bit inputs (`data0`, `data1`) based on a 16-bit instruction set.

### Technical Details:
* **Instruction Decoding:**
    * **Bits [15:14]:** Output location selection.
    * **Bits [13:12]:** Operation Type (Shift, Arithmetic, Logic, Special).
    * **Bits [11:10]:** Specific Operation selection.
* **Operations:**
    * **Shift:** Right/Left shift of `data0` by `data1`.
    * **Arithmetic:** Addition (with overflow detection) and Subtraction.
    * **Logic:** Bitwise AND, OR, XOR.
    * **Special:** Returns `1` if the MSB and LSB of both inputs are all equal.
* **Flags:** Includes a `zero_flag` (via `comp_eq` module) and an `overflow_flag` for sums exceeding 8 bits.

## 2. Caesar Cipher Encoder
**Directory:** `/Caesar_Cipher`

A hardware implementation of the Caesar Cipher algorithm, designed to shift an input character by a specific 4-bit offset.

### Technical Details:
* **Structure:** Features a dedicated **Preprocessing** module separate from the Top level.
* **Logic:** Uses 8-bit adders and subtractors with magnitude comparators to handle character wrap-around.
* **Hardware Mapping (xc7s50csga324-1):**
    * **Inputs:** `SW[7:0]` for the character and `SW[15:12]` for the offset.
    * **Output:** `LED[7:0]` for the encoded character.
* **Components:** Mux2_8b, Demux2_8b, and 8-bit arithmetic blocks.

---

## 3. ROM Lookup Table & Validation
**Directory:** `/ROM_Lookup_Table`

This project demonstrates the use of ROM memories as Lookup Tables (LUT) for 2-bit arithmetic operations.

### Technical Details:
* **ROM Modules:** `rom_add` and `rom_and` act as LUTs for 2-bit input operations.
* **Verification:** An `ok` signal is generated by comparing ROM outputs with standard hardware calculation blocks.
* **Consistency:** The `ok` signal remains constant at `1` to signal correct functionality.
* **Special Module:** Returns `1` if both input numbers have an equal number of set bits ('1').
* **Flags:** Overflow detection for 2-bit arithmetic sums.

## Simulation and Verification
All modules include testbenches following these specifications:
* **Timing:** Simulations run for **200 time units**.
* **Input Stimulus:** Waveform transitions are spaced at **5ns intervals**.
* **Toolchain:** Developed and simulated using **Xilinx Vivado 2025.2 XSim**.
* **Target Part:** `xc7s50csga324-1`.

---

## How to Use
1. Clone the repository.
2. Open the `.xpr` project files in Xilinx Vivado.
3. Run behavioral simulation to verify logic.
4. For the Caesar Cipher, use the provided constraints for the Spartan-7 board.

# Digital Design Projects - Verilog Implementations

This repository contains a collection of digital logic design projects implemented in **Verilog/SystemVerilog** using **Vivado 2025.2**. The projects range from arithmetic units to hardware-level encryption and ROM-based lookup tables.

All projects are targeted for the **Xilinx Spartan-7 FPGA (Part: xc7s50csga324-1)**.

---

## 1. 8-Bit Arithmetic Logic Unit (ALU)
**Directory:** `/ALU`

[cite_start]A simplified ALU that performs calculations between two 8-bit inputs (`data0`, `data1`) based on a 16-bit instruction set[cite: 7, 8, 9, 71].

### Technical Details:
* **Instruction Decoding:**
    * [cite_start]**Bits [15:14]:** Output location selection[cite: 47, 71].
    * [cite_start]**Bits [13:12]:** Operation Type (Shift, Arithmetic, Logic, Special)[cite: 45, 72].
    * [cite_start]**Bits [11:10]:** Specific Operation selection[cite: 44, 72].
* **Operations:**
    * [cite_start]**Shift:** Right/Left shift of `data0` by `data1`[cite: 11, 14, 73].
    * [cite_start]**Arithmetic:** Addition (with overflow detection) and Subtraction[cite: 18, 26, 78].
    * [cite_start]**Logic:** Bitwise AND, OR, XOR[cite: 29, 31, 33].
    * [cite_start]**Special:** Returns `1` if the MSB and LSB of both inputs are all equal[cite: 38, 75].
* [cite_start]**Flags:** Includes a `zero_flag` (via `comp_eq` module) and an `overflow_flag` for sums exceeding 8 bits[cite: 49, 68, 74, 78].

## 2. Caesar Cipher Encoder
**Directory:** `/Caesar_Cipher`

[cite_start]A hardware implementation of the Caesar Cipher algorithm, designed to shift an input character by a specific 4-bit offset[cite: 112, 169].

### Technical Details:
* [cite_start]**Structure:** Features a dedicated **Preprocessing** module separate from the Top level[cite: 118, 178].
* [cite_start]**Logic:** Uses 8-bit adders and subtractors with magnitude comparators to handle character wrap-around[cite: 175, 176, 177, 179].
* **Hardware Mapping (xc7s50csga324-1):**
    * [cite_start]**Inputs:** `SW[7:0]` for the character, `SW[15:12]` for the offset[cite: 219, 220].
    * [cite_start]**Output:** `LED[7:0]` for the encoded character[cite: 221].
* [cite_start]**Components:** Mux2_8b, Demux2_8b, and 8-bit arithmetic blocks[cite: 126, 157, 173, 174].

## 3. ROM Lookup Table & Validation
**Directory:** `/ROM_Lookup_Table`

[cite_start]This project demonstrates the use of ROM memories as Lookup Tables (LUT) for 2-bit arithmetic operations[cite: 259, 260].

### Technical Details:
* [cite_start]**ROM Modules:** `rom_add` and `rom_and` act as LUTs for 2-bit input operations[cite: 260, 265].
* [cite_start]**Verification:** An `ok` signal is generated by comparing ROM outputs with standard hardware calculation blocks[cite: 262]. [cite_start]The `ok` signal remains constant at `1`[cite: 263].
* [cite_start]**Special Module:** Returns `1` if both input numbers have an equal number of set bits ('1')[cite: 268].
* [cite_start]**Flags:** Overflow detection for 2-bit arithmetic sums[cite: 264].

## Simulation and Verification
All modules include testbenches following these specifications:
* [cite_start]**Timing:** Simulations run for **200 time units**[cite: 97, 286].
* [cite_start]**Input Stimulus:** Waveform transitions are spaced at **5ns intervals**[cite: 94, 284].
* [cite_start]**Toolchain:** Developed and simulated using **Xilinx Vivado 2025.2 XSim**[cite: 315, 340, 368].
* [cite_start]**Target Part:** `xc7s50csga324-1`.
